ΕΙΣΑΓΩΓΗ ΣΤΟ RANDOM TESTING

Το Random Testing είναι μια τεχνική black box testing η οποία βασίζεται στην παραγωγή τυχαίων εισόδων. Σε αυτή την περίπτωση τα προγράμματα θα ελέγχονται από ανεξάρτητα inputs που ούτε οι testers θα γνωρίζουν. Τα αποτελέσματα όπως και σε όλες τις εφαρμογές του software testing έτσι και εδώ θα συγκριθούν με τις προ απαιτούμενες προδιαγραφές που έχουν θέσει οι δημιουργοί του συστήματος για να επιβεβαιώσουν εάν ο έλεγχος είναι επιτυχής ή όχι. Η πραγματοποίηση ενός τέτοιου ελέγχου αρχικά θα γίνει όταν δεν θα μπορέσουν οι testers να εντοπίσουν σφάλματα σε κανονικό διάστημα ή στον χρόνο που πρέπει και για αυτό τον λόγο καταφεύγουν στα τυχαία inputs. Η χρησιμοποίηση τυχαίων εισόδων πολύ πιθανό να δημιουργήσει δεδομένα που οι testers δεν σκέφτηκαν να ελέγξουν είτε δεν διέθεταν τον χρόνο για να τεστάρουν inputs από όλες τις περιπτώσεις ελέγχου που έχουν θεσπίσει. Όπως και στις άλλες τεχνικές του testing έτσι και εδώ το χρειαζόμαστε για να εξασφαλιστεί τόσο η αξιοπιστία όσο και η απόδοση του software. Χάρη σε αυτό μειώνεται ο χρόνος που χρειάζονται οι δοκιμαστές για να βγάλουν ακριβή συμπεράσματα για την ποιότητα του προιόντος τους αλλά και οι προσπάθειες που κάνουν για να δούνε αν το λογισμικό είναι επιτυχή.

Στην εικόνα αποτυπώνεται ένα από τα πιο απλά παραδείγματα ελέγχου μια συνάρτησης του λογισμικού από μια γεννήτρια τυχαίων αριθμών. Οι διάφοροι αριθμοί θα εξεταστούν με την βοήθεια κάποιου προγράμματος στην προκειμένη περίπτωση του QuickCheck και για κάθε τυχαίο input που θα δοθεί θα εξάγουμε το αντίστοιχο output καθώς και τυχόν σφάλματα τα οποία θα καταγραφούν θα επιδιορθωθούν και μετά ο tester θα επαναλάβει την ίδια διαδικασία για τον επανέλεγχο.

ΒΗΜΑΤΑ ΥΛΟΠΟΙΗΣΗΣ

Για την εφαρμογή του random testing οι δοκιμαστές του λογισμικού πραγματοποιούν κάποια συγκεκριμένα βήματα.

  1. Ταυτοποίηση των τυχαίων εισόδων που πρόκειται να αξιολογήσουν το σύστημα προσπαθώντας να εντοπίσουν τα σφάλματά του.
  2. Επιλογή δεδομένων εισόδου ανεξάρτητα από τις περιπτώσεις ελέγχου που έχουν καθορίσει οι δοκιμαστές στο αντίστοιχο στάδιο του testing.
  3. Εκτέλεση του ελέγχου με τα τυχαία δεδομένα εισόδου.
  4. ΚΣύγκριση των αποτελεσμάτων με τα προσδοκόμενα outputs που έχουν προκαθοριστεί.
  5. Σε περίπτωση εντοπισμού σφάλματος ή σφαλμάτων άμεση επιδιόρθωση και επανέλεγχος για την επιβεβαίωση και επικύρωση της σωστής λειτουργίας του λογισμικού.

ΠΑΡΑΔΕΙΓΜΑ ΠΑΡΑΓΩΓΗΣ ΤΥΧΑΙΩΝ ΑΡΙΘΜΩΝ

Για την δημιουργία των τυχαίων αριθμών που θα αξιολογήσουν το λογισμικό υπάρχουν προγράμματα και εργαλεία που συμβάλλουν σε αυτό. Παρακάτω βλέπουμε ένα πολύ απλό παράδειγμα σε γλώσσα java που κατασκευάζει 10 τυχαίους αριθμούς από το 1 μέχρι το 100.

public class RandomNumbers{

ΠΛΕΟΝΕΚΤΗΜΑΤΑ

  • Στις περισσότερες περιπτώσεις ελέγχων χάρη στο χαρακτηριστικό δημιουργίας τυχαίων δεδομένων εισόδου τα σφάλματα που προκύπτουν μπορούν να εντοπιστούν εύκολα και γρήγορα.
  • Η εφαρμογή αυτής της τεχνικής δεν κοστίζει καθώς είναι εύκολα και γρήγορα εκτελέσιμη.
  • Ο σχεδιασμός του software testing απλοποιείται σε μεγάλο βαθμό καθώς είναι πολύ πιο γρήγορος και ανεξάρτητος από τις απαιτήσεις του συστήματος λόγω των τυχαίων δεδομένων που θα χρησιμοποιηθούν.
  • Δεν υπάρχει καμία πιθανότητα μεροληψίας από τον ελεγκτή καθώς τα δεδομένα εισόδου που θα εξεταστούν είναι τυχαία και ανεξάρτητα μεταξύ τους και δεν τα γνωρίζει ούτε ο ίδιος οπότε όσον αφορά την αξιοπιστία των δεδομένων μπορούμε να είμαστε απόλυτα βέβαιοι για το testing του προιόντος..
  • Τέλος το random testing δεν προϋποθέτει καμία γνώση προγραμματισμού ούτε του πως οι developers έχουν υλοποιήσει τον κώδικα παρά μόνο τις απαιτήσεις του συστήματος που έχουν θέσει και το αν τα αποτελέσματα συνάδουν με αυτές.
ΜΕΙΟΝΕΚΤΗΜΑΤΑ
  • Λόγω των τυχαίων εισόδων υπάρχουν συστήματα που αυτή η τεχνική οδηγεί σε μη ρεαλιστικά αποτελέσματα.
  • Κάποια από τα δεδομένα και τις περιπτώσεις που εξετάζονται μπορεί να είναι περιττά είτε λόγω των απαιτήσεων του συστήματος είτε επειδή έχουν ελεγχθεί ξανά.
  • Τα αποτελέσματα του random testing είναι και αυτά τυχαία επομένως σε πολλές περιπτώσεις δεν πρόκειται να πάρουμε τις ακριβείς προκαθορισμένες τιμές.
  • Ακριβή αποτελέσματα θα πάρουμε παρά μόνον στην περίπτωση όπου οι προδιαγραφές και οι απαιτήσεις του συστήματος δεν είναι ακριβείς.
  • Αυτή η τεχνική δεν συνδυάζει καμία άλλη από τις πολλές τεχνικές που διαθέτει το software testing και το black box testing για τον εντοπισμό σφαλμάτων.
